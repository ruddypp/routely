# Routely End-to-End QA E2E Plan

- [ ] QA E2E: write and run the local dashboard-first smoke report after Backend/Frontend local lifecycle slices land. Validate `routely` single command, first-run dashboard, app creation/editing, Start All for multiple enabled apps, disabled app skip, per-app stop/restart, logs, health, local routes/proxy status, and responsive views at 375px, 768px, and 1280px. Write a dated report under `docs/qa/` with screenshots/log links and do not commit it.

- [ ] QA E2E: write and run the one-VPS operations smoke report after production slices land. Validate install/run guidance, production auth, server doctor, one app deploy, domain/proxy/TLS status, env/secrets, logs, health, database/backups, notifications, and dashboard clarity. If real VPS/DNS/GitHub credentials are unavailable, mark those checks blocked with exact environment requirements. Write a dated report under `docs/qa/` and do not commit it.

- [ ] QA E2E: write and run the GitHub redeploy smoke report after GitHub slices land. Validate a successful branch push redeploys the intended app and a broken push leaves inspectable failure logs. Write a dated report under `docs/qa/` and do not commit it.
