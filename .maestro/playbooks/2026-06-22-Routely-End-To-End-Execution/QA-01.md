# QA Phase 01: Public Alpha End-To-End Validation

Playbook type: task-based. Each checkbox is self-contained and can run in a fresh agent context.

- [ ] QA E2E: read `AGENTS.md`, `docs/00-product-brief.md`, `docs/01-alpha-plan.md`, `docs/03-demo-acceptance-plan.md`, `docs/09-current-status.md`, `docs/13-end-to-end-execution-plan.md`, and the latest Backend/Frontend commits. Run the local dashboard-first demo from public docs only: clean workspace, `routely` bootstrap, register/import at least three apps and one database if supported, Start All, stop one app, disable one app, inspect logs/status/health, and validate responsive desktop/tablet/mobile behavior. Write a dated report to `docs/qa/end-to-end-local-demo-2026-06-22.md` with environment, commands, pass/fail evidence, screenshots if captured, blockers, and owner routing. Do not commit the report. Verification: report exists and clearly states whether the local demo passes.

- [ ] QA E2E: read `docs/03-demo-acceptance-plan.md`, `docs/13-end-to-end-execution-plan.md`, and current public setup docs. Validate production init/doctor, auth, app deploy, domain verification, proxy route, HTTPS/TLS truthfulness, env/secrets redaction, logs, deploy history, health, database, backup state, and deferred-state copy. If environment is unavailable, document the exact missing prerequisite and perform every local/static/API check possible. Write `docs/qa/end-to-end-vps-demo-2026-06-22.md`. Do not commit the report. Verification: report separates verified results from environment blockers.

- [ ] QA E2E: read `docs/03-demo-acceptance-plan.md`, `docs/13-end-to-end-execution-plan.md`, GitHub setup docs, and current implementation. If credentials are unavailable, document the exact missing prerequisite and run route/static checks possible. Write `docs/qa/end-to-end-github-demo-2026-06-22.md`. Do not commit the report. Verification: report states pass/fail, evidence, and owner-routed findings.
